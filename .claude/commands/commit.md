# /commit

変更内容を整理し、適切なコミットメッセージで `git commit` を行います。

## 利用方法

```
/commit
```

## 手順

1. `git status` と `git diff --stat` を実行し、変更内容を把握します。
2. 変更を論理的な単位に分割できる場合は、ユーザーに確認せずに staging を調整します。
3. 日本語で簡潔なコミットメッセージを作成します。形式は以下を参考にします。
   - 1行目: 変更の要約（50文字程度）
   - 2行目: 空行
   - 3行目以降: 必要に応じた詳細
4. `git commit` を実行します。

## 禁止事項

- ユーザーに確認を取らずに `git push` は行いません。
- `--amend` や `git reset` 等の履歴改変は、明示的な指示がない限り行いません。
